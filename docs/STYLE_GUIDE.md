# Python style guide

This project follows these conventions:

- Use [PEP 8](https://peps.python.org/pep-0008/) as the baseline.
- Run `ruff` for linting and `black` for formatting.
- Use type annotations where practical.
- Functions should be short and descriptive; prefer composition over large monoliths.
- Write docstrings for public functions and classes.

Apply these guidelines consistently when contributing new modules or scripts.
